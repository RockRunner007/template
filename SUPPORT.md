# Support

## Getting Help

If you need help with this project, here are several ways to get support:

### Documentation

Start with our documentation:

- [README](README.md) — Quick start and project overview
- [Contributing Guide](CONTRIBUTING.md) — How to contribute to the project
- [Security Policy](SECURITY.md) — How to report security vulnerabilities
- `docs/runbooks/` — Operational guides for common tasks
- `prompts/` — AI-friendly prompts for project context

### Reporting Issues

Found a bug or have a feature request?

1. **Before opening an issue:** Check if the issue already exists by searching our [GitHub Issues](https://github.com/RockRunner007/template/issues)
2. **Create a new issue** with:
   - Clear title describing the problem or feature
   - Description with context and steps to reproduce (for bugs)
   - What you expected to happen and what actually happened
   - Your environment details if relevant

### Discussions

For questions, ideas, or general discussions:

- Use [GitHub Discussions](https://github.com/RockRunner007/template/discussions)
- This is a great place for brainstorming and community support

### Contacting Maintainers

- Primary maintainer: @stevencarlson
- For security concerns: See [SECURITY.md](SECURITY.md)

## Response Times

We aim to respond to issues and discussions within 3 business days. However, response times may vary depending on issue complexity and maintainer availability.

## Community Guidelines

- Be respectful and constructive
- Assume good intent
- See our [Code of Conduct](CODE_OF_CONDUCT.md) for community standards

## Contributing a Fix

Interested in fixing an issue yourself? Check out our [Contributing Guide](CONTRIBUTING.md) for steps to fork, branch, and submit a pull request.
